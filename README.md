# Программный комплекс для прогнозирования численности населения

Данный проект представляет собой программный комплекс для проведения вычислительных экспериментов по прогнозированию численности населения на основе балансовой модели притока и оттока.

Комплекс разрабатывался как исследовательский инструмент: он позволяет выбрать страну, задать параметры прогноза, построить прогноз по одному из реализованных методов, сравнить результат с фактическими данными и, при необходимости, с прогнозными сценариями ООН WPP. Отдельное внимание уделено удобному запуску проекта, автоматической подготовке данных и сохранению результатов экспериментов.

Проект реализован на Python. Пользовательский интерфейс построен на Streamlit.

---

## Основные возможности

Программный комплекс позволяет:

- автоматически подготовить рабочее Python-окружение;
- автоматически загрузить файл демографических показателей WPP с сайта ООН;
- сформировать локальный CSV-файл с историческими демографическими данными;
- выбрать страну или регион из набора данных ООН;
- построить прогноз численности населения на заданный период;
- использовать один из нескольких методов прогнозирования;
- задавать параметры окна аппроксимации и окна экстраполяции;
- выбирать способ учёта миграции в балансовой модели;
- включать или отключать сравнение с прогнозными сценариями ООН;
- рассчитывать метрики качества прогноза;
- просматривать графики населения, притока и оттока;
- анализировать динамику ошибок по годам;
- сохранять результаты экспериментов;
- повторно запускать ранее сохранённые эксперименты с теми же параметрами;
- выгружать подробные результаты по прогнозным окнам в Excel.

## Идея модели

В основе комплекса лежит балансовое представление демографической динамики. Численность населения рассматривается как величина, изменяющаяся под действием двух потоков:

- притока;
- оттока.

В качестве исходных демографических показателей используются:

- численность населения на начало года;
- число рождений;
- число смертей;
- чистая миграция.

После предварительной обработки для каждого года рассчитываются приток, отток и фактическая численность населения на конец года.

В общем виде используется соотношение:

```text
N(t) = N(0) + I(t) - O(t)
```

где:

- `N(t)` - численность населения;
- `N(0)` - базовая численность населения;
- `I(t)` - интегральный приток;
- `O(t)` - интегральный отток.

Такая постановка позволяет прогнозировать не только саму численность населения, но и промежуточные компоненты модели: годовой приток, годовой отток, интегральные ряды и их процентные приросты.

## Реализованные методы прогнозирования

В проекте реализованы три основных метода.

### 1. Прямой прогноз потоков

Метод `inflow_outflow_direct`.

В этом варианте прогнозируются интегральные ряды притока и оттока. После этого из прогнозных интегральных значений восстанавливаются годовые притоки, годовые оттоки и итоговая численность населения.

Метод является наиболее прямым, так как работает непосредственно с накопленными значениями потоков.

---

### 2. Темпы прироста интегралов

Метод `integrated_growth_rate`.

В этом варианте сначала рассчитываются процентные приросты интегральных рядов притока и оттока. Далее прогнозируются уже не сами интегральные ряды, а их темпы прироста.

После получения прогнозных темпов восстанавливаются:

- интегральный приток;
- интегральный отток;
- годовой приток;
- годовой отток;
- численность населения.

---

### 3. Скорость темпов

Метод `growth_rate_speed`.

В этом варианте дополнительно рассчитываются скорости изменения процентных приростов интегральных рядов. Прогноз строится через динамику изменения темпов роста.

Если при восстановлении темпов возникают некорректные значения, алгоритм переходит к резервному варианту прогнозирования процентных приростов
Такой вариант помогает сохранить устойчивость расчёта

---

## Аппроксимация временных рядов

Для прогнозирования временных рядов используются два типа аппроксимации:

- линейная;
- степенная.

Также доступен автоматический режим.

В автоматическом режиме степенная аппроксимация применяется только для положительных рядов
Если ряд содержит нулевые или отрицательные значения, программа использует линейную аппроксимацию, поскольку степенная модель строится через логарифмирование

---

## Режим скользящего окна

В интерфейсе доступна настройка:

```text
Использовать СОА
```

СОА - это режим рекуррентного скользящего окна внутри каждого шага прогноза.

Если режим выключен, модель строится один раз по окну аппроксимации и сразу прогнозирует весь блок экстраполяции.

Если режим включен, прогноз строится пошагово:

1. по текущему окну строится прогноз на один год;
2. полученное прогнозное значение добавляется в конец окна;
3. первое значение окна удаляется;
4. модель перестраивается;
5. процесс повторяется до конца горизонта.

Такой режим позволяет учитывать изменение локальной динамики ряда в процессе построения прогноза.

## Данные

Проект использует данные World Population Prospects.

Основным исходным файлом является Excel-файл вида:

```text
WPP*_GEN_F01_DEMOGRAPHIC_INDICATORS_FULL.xlsx
```

При первом запуске проекта файл автоматически скачивается с сайта ООН и сохраняется в каталог `data`.

Из этого файла формируется локальный CSV-файл:

```text
data/population_data.csv
```

CSV-файл создаётся на основе листа `Estimates`. Служебная шапка Excel-файла удаляется, а данные сохраняются в табличном виде.

Ожидаемая структура CSV соответствует структуре файла WPP и содержит, в частности, следующие поля:

```text
Variant
Region, subregion, country or area *
Year
Total Population, as of 1 January (thousands)
Births (thousands)
Total Deaths (thousands)
Net Number of Migrants (thousands)
```

Внутри программы значения из WPP переводятся из тысяч человек в абсолютные значения.

## Сравнение с прогнозами ООН

Программный комплекс может сравнивать собственный прогноз с прогнозными сценариями ООН.

Для этого используются листы Excel-файла WPP, соответствующие прогнозным сценариям. Например:

- `Medium variant`;
- `High variant`;
- `Low variant`;
- `Zero-migration`;
- другие сценарии, если они присутствуют в файле WPP.

Список сценариев формируется динамически по листам Excel-файла
Проект не привязан только к WPP2024 и может работать с более новыми версиями файла, если сохраняется общий формат данных

Прогноз ООН не используется при построении собственного прогноза модели
Он подключается после расчёта и используется как внешний ориентир для сравнения

---

## Метрики качества

В проекте рассчитываются следующие метрики:

- `MAPE` - средняя абсолютная процентная ошибка;
- `Точность` - показатель вида `100% - абсолютная процентная ошибка`;
- `SMAPE` - симметричная средняя абсолютная процентная ошибка;
- `RMSE` - среднеквадратичная ошибка;
- `MAE` - средняя абсолютная ошибка.

Если сравнение с WPP включено, метрики отображаются раздельно:

- отдельно по фактическим значениям;
- отдельно по выбранному сценарию ООН.

Если фактические значения для будущих лет отсутствуют, сравнение с ООН позволяет получить внешний ориентир для оценки прогноза.

## Назначение основных файлов

### `app.py`

Основной файл пользовательского интерфейса. В нём реализовано Streamlit-приложение: боковая панель с параметрами, запуск расчёта, вкладки с результатами, графики, таблицы и сохранение экспериментов.

---

### `launch.pyw`

Графический модуль запуска для Windows

Он выполняет первичную инициализацию проекта:

1. проверяет наличие `.venv`;
2. создаёт виртуальное окружение, если его нет;
3. устанавливает программные зависимости;
4. проверяет наличие данных;
5. скачивает WPP-файл, если он отсутствует;
6. формирует `population_data.csv`;
7. запускает Streamlit-приложение.

Файл запускается без консольного окна.

---

### `launch.py`

Аналогичный сценарий запуска для систем, где удобнее использовать обычный Python-скрипт

---

### `scripts/bootstrap_data.py`

Скрипт подготовки данных.

Он отвечает за поиск и загрузку файла WPP, проверку его структуры и формирование локального CSV-файла для приложения.

---

### `src/data_processing/storage.py`

Модуль загрузки и предварительной обработки исторических данных.

Он формирует ряды:

- `population`;
- `births`;
- `deaths`;
- `net_migration`;
- `inflow`;
- `outflow`;
- `total_actual`.

---

### `src/data_processing/un_forecast.py`

Модуль работы с прогнозными сценариями ООН.

Он ищет доступный WPP-файл, определяет реальные прогнозные сценарии, загружает выбранный сценарий и преобразует его к структуре, совместимой с результатами собственной модели.

---

### `src/forecasting`

Каталог с реализацией методов прогнозирования.

Все методы имеют общий интерфейс:

```python
fit(data)
predict(horizon)
```

Такой интерфейс упрощает добавление новых методов и запуск разных алгоритмов через единый механизм эксперимента

---

### `src/experiment/runner.py`

Модуль запуска вычислительного эксперимента.

Он связывает между собой:

- конфигурацию;
- данные;
- выбранный метод;
- прогноз по окнам;
- расчёт метрик;
- сравнение с WPP.

---

### `src/experiment/saver.py`

Модуль сохранения результатов.

Он сохраняет:

- конфигурацию эксперимента;
- метрики;
- прогнозный ряд;
- полную таблицу;
- детальные результаты по окнам.

---

## Установка и запуск

### Рекомендуемый способ запуска на Windows

Для обычного запуска достаточно открыть файл:

```text
launch.pyw
```

При первом запуске он сам подготовит проект. Пользователю не нужно вручную создавать виртуальное окружение, устанавливать библиотеки или скачивать данные.

После инициализации модуль запуска автоматически откроет веб-интерфейс приложения

---

### Запуск через Python

Если используется Linux, macOS или запуск из терминала, можно выполнить:

```bash
python launch.py
```

---

### Ручной запуск

Если окружение уже подготовлено, приложение можно запустить вручную:

```bash
streamlit run app.py
```

или:

```bash
python -m streamlit run app.py
```

---

## Что происходит при первом запуске

При первом запуске выполняются следующие шаги:

1. Проверяется наличие виртуального окружения `.venv`.
2. Если `.venv` отсутствует, оно создаётся автоматически.
3. Устанавливаются программные зависимости из `requirements.txt`.
4. Проверяется каталог `data`.
5. Если WPP-файл отсутствует, он скачивается.
6. Из WPP-файла формируется `population_data.csv`.
7. Запускается Streamlit-приложение.

В графическом запускателе каждый этап отображается через прогресс-бары и текстовый журнал.

---

## Повторный запуск

После первой инициализации запуск становится быстрее.

Если `.venv` уже существует, программные зависимости не устанавливаются заново. Если данные уже подготовлены, WPP-файл повторно не скачивается, а CSV не пересобирается без необходимости.

В этом случае модуль запуска открывает веб-интерфейс приложения

---

## Работа без автоматического скачивания

Если автоматическое скачивание невозможно, например из-за отсутствия доступа к интернету, файл WPP можно положить вручную в каталог:

```text
data/
```

Имя файла должно соответствовать шаблону:

```text
WPP*_GEN_F01_DEMOGRAPHIC_INDICATORS_FULL.xlsx
```

После этого нужно снова запустить `launch.pyw` или `launch.py`. Скрипт найдёт файл и сформирует `population_data.csv`.

---

## Сохранение результатов

После построения прогноза результаты можно сохранить на вкладке `Эксперименты`.

В каталог `outputs` записываются файлы:

```text
*_config.json
*_metrics.json
*_forecast.csv
*_full_table.csv
```

Также на вкладке `Детали по окнам` можно сохранить подробные результаты в Excel.

Сохранённый эксперимент можно загрузить повторно
При загрузке комплекс заново выполняет расчёт с теми же параметрами, поэтому старые конфигурации удобно использовать для проверки обновлённой версии проекта

---

## Добавление нового метода прогнозирования

Чтобы добавить новый метод, нужно:

1. Создать новый файл в каталоге:

```text
src/forecasting/
```

2. Реализовать класс метода.

Если метод основан на балансовой модели притока и оттока, удобнее наследоваться от `InflowMethod`.

3. Реализовать методы:

```python
fit(data)
predict(horizon)
```

4. Добавить новый метод в словарь выбора методов в `app.py` и `runner.py`.

Такой подход позволяет расширять комплекс без изменения общей архитектуры проекта.

## Возможные проблемы и решения

### Долго выполняется первый запуск

Первый запуск может занимать заметное время, потому что создаётся виртуальное окружение, устанавливаются программные зависимости и скачивается большой Excel-файл WPP.

Повторные запуски должны выполняться быстрее.

---

### WPP-файл не скачивается

Возможные причины:

- нет доступа к интернету;
- изменилась структура сайта ООН;
- файл временно недоступен.

Решение: скачать файл WPP вручную и положить его в каталог `data`.

---

### Не отображается выбранный сценарий ООН

Список сценариев формируется по листам реального WPP-файла. Если в новой версии файла нет сценария, который был сохранён в старой конфигурации, приложение выберет доступный сценарий и покажет предупреждение.

---

### Нет фактических данных для оценки

Если прогноз строится за пределы исторического периода, фактические значения отсутствуют. В этом случае для оценки можно включить сравнение с прогнозом ООН.

---

### Проблемы с кодировкой в запускателе

В проекте принудительно используется UTF-8 для дочерних Python-процессов. Если в журнале запускателя всё равно появляются некорректные символы, стоит проверить системную кодировку и версию Python.

Рекомендуется использовать Python 3.10 или новее.
